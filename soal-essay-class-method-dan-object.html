3. suatu method yang dapat dijalankan otomatis pada saat object dari class dibuat, dikenal dengan


Metode yang dapat dijalankan secara otomatis saat object dari class yang telah dibuat disebut dengan constructor. onstructor biasanya digunakan untuk menginisialisasi atau memberikan nilai (melalui parameter) kepada sebuah objek pada saat objek pertama kali diciptakan.

Pembahasan:

Constructor merupakan method yang dijalankan pertama kali pada saat sebuah objek pertama kali dibuat. Adapun yang menjadi perbedaan method constructor dengan method lainnya pada java adalah nama method constructor wajib sama dengan nama kelasnya. Constructor biasanya digunakan untuk menginisialisasi atau memberikan nilai (melalui parameter) kepada sebuah objek pada saat objek pertama kali diciptakan.

Di sisi lain, selain constructor, ada juga yang namanya destructor. Destructor adalah method yang khusus untuk yang akan dieksekusi saat objek akan dihapus dari memori sistem. Java sendiri tidak mempunyai method destructor, hal ini dikarena Java menggunakan sistem garbage collector untuk pengaturan memori sistemnya. Jadi yang menjadi garbage collector akan menghapus objek yang tidak digunakan secara otomatis.

12. Jelaskan yang dimaksud dengan Class dan Object, berikan juga contohnya


Class atau kelas adalah sekelompok objek yang memiliki kesamaan. Sedangkan, object atau objek adalah suatu entitas baik real maupun abstrak yang dapat diidentifikasi dan direpresentasikan. Dengan kata lain, object juga merupakan bagian dari suatu class.  

Pembahasan

Class dan Object adalah dua hal penting dalam pemrograman, terutama pada pemrograman berbasis objek. Dalam suatu kelas yang terdapat objek, objek tersebut memiliki kesamaan dari sisi macam atribut dan operasi, meskipun nilai atribut sangat mungkin berbeda. Sehingga, suatu objek tentu memiliki sekumpulan atribut dan operasi.  Untuk mempermudah pemahaman, berikut salah satu ilustrasinya:

Objek buku memiliki nama kamus bahasa Inggris, memiliki beberapa atribut seperti jumlah halaman, warna kamus, pengarang dan penerbit, nomor registrasi, tanggal terbit dan sebagainya. Operasi yang dilakukan pada kamus tersebut seperti, mengubah jumlah halaman, mengubah harga, mengubah nomor registrasi dan sebagainya. Operasi ini hanya akan mengubah nilai atribut dan tidak mengubah jenis atribut itu sendiri. Selanjutnya, sekumpulan objek buku tersebut dapat digolongkan menjadi suatu kelas.

18. object diagram dan class diagram ???


Objek diagram adalah suatu diagram yang berfungsi untuk mengatur atribut,objek dan hubungan antara contoh dalam diagram, diagram objek juga dapat menampilkan struktur model system dalam waktu tertentu. Objek diagram berasal dari kelas diagram, sehingga diagram objek tergantung pada diagram kelas.

Dalam Unified Modeling Language (UML), diagram objek berfokus pada beberapa set tertentu dari objek dan atribut. Diagram objek dan diagram kelas berkaitan erat dan menggunakan notasi yang hampir sama. Kedua diagram dimaksudkan untuk memvisualisasikan struktur statis dari suatu sistem. Sementara diagram kelas menunjukkan kelas, diagram objek menampilkan contoh kelas (objek). diagram objek yang lebih konkret dari diagram kelas. Mereka sering digunakan untuk memberikan contoh atau bertindak sebagai kasus uji untuk diagram kelas.

Elemen Diagram Objek

Elemen dalam diagram objek sangat sederhana, elemen-elemen tersebut digambarkan oleh persegi panjang dan dihubungkan dengan garis. Dibawah ini merupakan elemen-elemen dari diagram objek :

   Objek

   Merupakan contoh dari kelas. Misal jika “mobil” adalah kelas, maka Toyota Avanza merupakan objek dari kelas tersebut.

   Judul Kelas

   Merupakan atribut tertentu dari kelas,

   Atribut Kelas

   Atribut persegi panjang dengan dua tab yang menunjukkan dari elemen software.

   Link

   Garis yang menghubungkan satu garis dengan garis yang lainnya.

Penggunaan Diagram Objek

   Mendeskripsikan bagaimana kumpulan objek tertentu saling berhubungan.

   Mengujikan diagram kelas dengan menggunakan diagram objek tertentu.

   Memeriksa iterasi tertentu dari suatu sistem.

   Mendapatkan gambaran dengan resolusi yang tinggi dari suatu sistem yang akan anda kembangkan.

   Membuat prototype

   Pemodelan struktur data yang kompleks

   Memahami sistem dari prespektif yang praktis.

Langkah-Langkah Membuat Diagram Objek

Object diagram tunggal tidak dapat menangkap semua kasus yang diperlukan atau lebih tepatnya tidak dapat menentukan semua objek dari suatu sistem. Untuk mengatasi hal tersebut maka ada beberapa tahapan yg perlu dilakukan :

   Menganalisis sistem dan memutuskan kasus yang memiliki data penting dan asosiasi.

   Mempertimbangkan hanya contoh-contoh yang akan mencakup fungsi tersebut.

   Membuat optimasi beberapa seperti jumlah contoh yang terbatas.

Hal-hal yang perlu diingat dan dipahami sebelum menggambar Object diagram:

   Diagram objek yang terdiri dari benda-benda.

   Link dalam diagram objek digunakan untuk menghubungkan obyek.

   Obyek dan link adalah dua elemen yang digunakan untuk membangun sebuah diagram objek.

Hal-hal berikut ini harus diputuskan sebelum memulai pembangunan diagram:

   Diagram objek harus memiliki nama yang berarti untuk menunjukkan tujuannya.

   Unsur-unsur yang paling penting adalah untuk diidentifikasi.

   Asosiasi antara objek-objek yang harus diklarifikasi.

   Nilai dari elemen yang berbeda harus diambil untuk memasukkan dalam diagram objek.

   Menambahkan catatan yang tepat pada titik-titik di mana kejelasan lebih diperlukan.

Berikut adalah implementasi dari class Matakuliah:

public class Matakuliah {

private String kode;

private String nama;

private int sks;

private double nilai;

public Matakuliah(String kode, String nama, int sks, double nilai) {

this.kode = kode;

this.nama = nama;

this.sks = sks;

this.nilai = nilai;

}

public String getKode() {

return kode;

}

public void setKode(String kode) {

this.kode = kode;

}

public String getNama() {

return nama;

}

public void setNama(String nama) {

this.nama = nama;

}

public int getSks() {

return sks;

}

public void setSks(int sks) {

this.sks = sks;

}

public double getNilai() {

return nilai;

}

public void setNilai(double nilai) {

this.nilai = nilai;

}

@Override

public String toString() {

return String.format("%-10s%-20s%-5d%.2f", kode, nama, sks, nilai);

}

}

Berikut adalah implementasi dari program nomor 2:

import java.util.ArrayList;

import java.util.Iterator;

public class Transkrip {

public static void main(String[] args) {

ArrayList<Matakuliah> transkrip = new ArrayList<>();

// Menambahkan beberapa Matakuliah ke dalam ArrayList

transkrip.add(new Matakuliah("IF001", "Pemrograman Java", 4, 3.5));

transkrip.add(new Matakuliah("IF002", "Basis Data", 3, 3.8));

transkrip.add(new Matakuliah("IF003", "Sistem Operasi", 3, 4.0));

transkrip.add(new Matakuliah("IF004", "Struktur Data", 4, 3.7));

// Menampilkan elemen-elemen transkrip menggunakan iterator

Iterator<Matakuliah> iterator = transkrip.iterator();

System.out.printf("%-10s%-20s%-5s%s%n", "Kode", "Nama", "SKS", "Nilai");

while (iterator.hasNext()) {

Matakuliah mk = iterator.next();

System.out.println(mk);

}

}

}

Kode Nama SKS Nilai

IF001 Pemrograman Java 4 3.50

IF002 Basis Data 3 3.80

IF003 Sistem Operasi 3 4.00

IF004 Struktur Data 4 3.70

37. buatlah konsep oop yang terdiri dari :class object methodpackageProsedur dan fungsi properti dari classinheritence pada class polymorphism pada class​


erikut adalah konsep OOP yang terdiri dari class, object, method, package, prosedur, dan fungsi properti dari class, inheritance pada class, dan polymorphism pada class:

Class

Class merupakan sebuah blueprint atau rancangan yang digunakan untuk membuat objek-objek dalam OOP. Class digunakan untuk menggambarkan suatu jenis objek, seperti misalnya class "Mobil" yang digunakan untuk membuat objek mobil.

Object

Objek merupakan sebuah instance dari class. Objek dibuat berdasarkan class dan memiliki karakteristik yang sama dengan class tersebut. Objek memiliki sifat-sifat yang unik yang tidak dimiliki oleh objek lainnya.

Method

Method adalah sebuah fungsi yang terdapat di dalam class dan digunakan untuk memproses data atau melakukan suatu tindakan pada objek. Method bisa diakses oleh objek yang dibuat dari class tersebut.

Package

Package adalah sebuah kumpulan class yang dikelompokkan berdasarkan fungsinya. Package digunakan untuk memudahkan pengelolaan program yang besar.

Prosedur dan Fungsi Properti dari Class

Prosedur adalah sebuah blok kode yang digunakan untuk melakukan suatu tindakan pada objek atau program. Sedangkan fungsi properti adalah sebuah blok kode yang digunakan untuk mengambil atau mengubah nilai dari properti yang terdapat pada suatu objek.

Inheritance pada Class

Inheritance adalah sebuah konsep OOP yang memungkinkan sebuah class untuk mewarisi sifat dan perilaku dari class yang lain. Class yang mewarisi disebut subclass, sedangkan class yang diwarisi disebut superclass.

Polymorphism pada Class

Polymorphism adalah sebuah konsep OOP yang memungkinkan sebuah objek untuk memiliki banyak bentuk atau perilaku. Polymorphism digunakan untuk membuat program yang lebih fleksibel dan mudah diubah, karena objek yang berbeda dapat diolah dengan cara yang sama. Contohnya adalah sebuah class "Hewan" yang memiliki subclass "Kucing" dan "Doggy", yang memiliki perilaku yang berbeda meskipun keduanya adalah hewan.
